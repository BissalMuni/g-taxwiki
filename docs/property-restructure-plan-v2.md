## 설계 원칙 (확정 후보)

1. **백본 = 책 A의 주제별 편/장.** 종합·상시적이라 뼈대로 적합.
2. **책 B Ⅳ 운영요령은 이미 주제별 그룹** → 책 A의 대응 장 안으로 흡수 (거의 1:1 정렬).
3. **책 B Ⅱ 유권해석** → 각 주제 단원 안 "사례/해석" 섹션으로 흡수.
4. **책 B Ⅰ 2026 개정사항** → **본문을 각 주제 단원으로 분산만.** (별도 요약 인덱스 리프 없음 — 위키는 매년 갱신되므로 "연도 편/리프"를 두지 않고 상시 주제트리에 녹임. ★확정)
5. **분리과세 25개** → 토지 아래 **저율 / 고율 / 정책목적 / 운영(신청·평가)** 4그룹. (★확정)
6. 4장(과표·세율)은 1차 plan처럼 **과세표준 / 세율 두 장으로 분리.**
7. **병합 방침 (★확정).** 행안부 항목은 기본적으로 **기존 주제 리프의 `■ 섹션`으로 흡수.** 양이 비대할 때(분리과세 25개)만 그룹 리프로 승격. 작은 서울 항목(소액징수면제·납세관리인 등)도 인접 리프 섹션으로 병합 → 리프 폭발 방지.
8. **출처 추적 필수 (★확정).** 섹션으로 흡수해도 **어느 책에서 왔는지 항상 추적 가능**해야 함. 아래 「출처 추적 규칙」 참조.

## 출처 추적 규칙 (★확정)

섹션으로 흡수해도 출처를 잃지 않기 위한 단일 규칙.

- **책 코드**: `[A]` = 서울 「재산세 운용실무」, `[B]` = 행안부 「부과·징수 운영요령」.
  - `[B]`는 항목번호까지: `[B37]`(운영요령 37번), `[B Ⅰ-법1]`(개정사항-지방세법 1번), `[B Ⅱ-3]`(유권해석 3번), `[B Ⅲ-1]`(세부담상한 계산 1번).
- **설계 단계(이 문서)**: 아래 트리에서 **리프의 각 `■ 섹션`마다 출처 태그**를 붙임 = 단일 진실(SSOT).
- **콘텐츠 단계(별도 대화)**: 두 군데에 출처를 박는다.
  1. 리프 `meta.sources` 배열 — 예: `sources: ["A", "B37", "B38", "B Ⅰ-법1"]`.
  2. 흡수된 `■ 섹션` 제목/말미에 출처 표기 — 예: `<CalcBox title="■ 신탁재산 납세의무자 (출처: 행안부 B37)">` 또는 섹션 끝 작은 라벨.
- 따라서 한 리프가 두 책을 섞어도, 메타·섹션 라벨로 **문장 단위까지 어느 책인지 추적** 가능.
